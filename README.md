# BULK MODE V6 — برنامج التضخيم النهائي 💪

تطبيق ويب يعمل أوفلاين (PWA) لمتابعة برنامج تضخيم العضلات لمدة 12 أسبوع، مع نظام تتبع احترافي يضاهي Hevy و Strong.

## ✨ المميزات

### الأساسيات
- 📱 **يعمل أوفلاين بالكامل** — بعد التثبيت لا يحتاج إنترنت
- 🗄️ **IndexedDB** — تخزين قوي ومنظّم، استيعاب أكبر بكثير من LocalStorage
- 💾 **تصدير/استيراد JSON** — احتفظ بنسخة احتياطية متكاملة
- 🌙 **تصميم RTL** — مصمّم خصيصاً للقراءة العربية
- 🔒 **خصوصية ١٠٠٪** — كل بيانة على جهازك، لا backend، لا cookies، لا تتبع

### تتبع التمارين (المستوى ١ — V5)
- ▶️ **جلسات تمرين كاملة** — زر "بدء الجلسة" أعلى كل يوم، مع شريط نشط يعرض المدة وعدد السيتات
- 🏆 **اكتشاف الأرقام القياسية تلقائياً** (Weight / Reps / Volume / 1RM Epley) — مع احتفال بصري + اهتزاز قوي
- ⏱ **مؤقت راحة ذكي** — يبدأ تلقائياً بعد كل سيت، يستمر عند تبديل التابات، إشعار صوتي + اهتزاز عند الانتهاء، أزرار **تخطي** و **+30s**
- 📊 **رسوم بيانية للتقدّم** (Chart.js):
  - تطوّر الوزن و 1RM المقدّر لكل تمرين عبر الزمن
  - الحجم الأسبوعي الإجمالي
  - تطور وزن الجسم
- 📜 **سجل جلسات كامل** — كل جلسة مع المدة، الحجم، السيتات، والـ PRs
- 🏆 **صفحة "أرقامي القياسية"** — مرجع سريع لأفضل أرقامك لكل تمرين
- 🎯 **شاشة ملخص نهاية الجلسة** — المدة، الحجم، السيتات، الـ PRs المحطّمة

### البدائل الذكية (V6)
- ⇄ **زر "بديل؟"** بجانب كل تمرين — يفتح Bottom Sheet بـ٣-٥ بدائل مرتّبة حسب تطابق العضلة
- ⚡ **تحذير وقت الذروة** (6-9م) — يبرز البدائل الأقل ازدحاماً
- 💡 **اقتراح ذكي:** إذا استبدلت نفس التمرين ٣ مرات → يقترح جعل البديل دائماً
- 🔄 **استبدال مرئي + تتبّع:** السيت يُحفَظ تحت اسم البديل، مع زر سريع لاستعادة الأصلي
- 📋 **كل الأجهزة (٢٤)** كخيار احتياطي — "ما لقيت اللي يناسبك؟"

## 🚀 رفع الموقع على GitHub Pages — خطوة بخطوة

### الخطوة ١: أنشئ مستودع جديد على GitHub

١. اذهب إلى [github.com/new](https://github.com/new)  
٢. **Repository name:** `bulkmode` (أو أي اسم تريده)  
٣. اجعله **Public** (شرط GitHub Pages المجاني)  
٤. اضغط **Create repository**

### الخطوة ٢: ارفع الملفات

**الطريقة الأسهل — عبر الواجهة (بدون terminal):**

١. في صفحة المستودع الجديد، اضغط **"uploading an existing file"**  
٢. اسحب جميع الملفات والمجلد التالي:
```
bulkmode/
├── index.html
├── manifest.json
├── service-worker.js
├── README.md
└── icons/
    ├── icon-192.png
    └── icon-512.png
```
٣. **مهم:** اسحب مجلد `icons` كاملاً (ليس كل صورة على حدة)  
٤. اكتب رسالة commit مثل: `Initial deploy`  
٥. اضغط **Commit changes**

**أو عبر terminal (إذا كنت معتاد):**

```bash
git init
git add .
git commit -m "Initial deploy"
git branch -M main
git remote add origin https://github.com/USERNAME/bulkmode.git
git push -u origin main
```

### الخطوة ٣: فعّل GitHub Pages

١. في صفحة المستودع، اذهب إلى **Settings** (الأيقونة الترسية)  
٢. من القائمة الجانبية، اختر **Pages**  
٣. تحت **Source**، اختر **Deploy from a branch**  
٤. تحت **Branch**، اختر **main** و **/(root)** ثم **Save**  
٥. انتظر ١-٢ دقيقة، ستظهر رسالة:
```
Your site is live at https://USERNAME.github.io/bulkmode/
```

### الخطوة ٤: ثبّت التطبيق على جوالك

**على Android (Chrome / Edge / Samsung Internet):**

١. افتح الرابط على جوالك  
٢. ستظهر رسالة "إضافة إلى الشاشة الرئيسية" — اضغط عليها  
٣. أو من قائمة المتصفح (⋮) → **"تثبيت التطبيق"** أو **"إضافة إلى الشاشة الرئيسية"**  
٤. أيقونة BULK MODE ستظهر على شاشتك الرئيسية كأي تطبيق

**على iOS (Safari فقط — Chrome على iOS لا يدعم PWA):**

١. افتح الرابط في **Safari** (مهم — ليس Chrome)  
٢. اضغط زر المشاركة (المربع مع السهم لأعلى)  
٣. مرّر لأسفل → **"إضافة إلى الشاشة الرئيسية"**  
٤. اضغط **إضافة**

بعد التثبيت، يعمل التطبيق ١٠٠٪ أوفلاين بدون إنترنت.

## 📊 كيف تستخدم نظام التتبع

### تسجيل جلسة كاملة

١. اذهب إلى تاب **"التمارين"** واختر يوم التمرين  
٢. اضغط **▶ بدء الجلسة** أعلى اليوم — سيظهر شريط أحمر بالأعلى يعدّ الوقت  
٣. تحت كل سيت ستجد حقلين: **كجم** و **تكرار** — أدخل ما حقّقته  
٤. اضغط **حفظ** — السيت يصير أخضر ✓، ومؤقت الراحة يبدأ تلقائياً  
٥. **🏆 رقم قياسي؟** التطبيق يكتشفه ويعرض احتفالاً بصرياً مع اهتزاز قوي  
٦. عند الانتهاء، اضغط **إنهاء** من الشريط العلوي — ستظهر شاشة ملخص بكل ما أنجزته

### مؤقت الراحة

- يبدأ **تلقائياً** بعد كل سيت محفوظ، بمدة مناسبة (٦٠ث للأزواج، ٩٠ث للـ SOLO)
- يعمل حتى لو بدّلت تاب أو خرجت من التطبيق (إشعار + اهتزاز عند الانتهاء)
- أزرار **تخطي** أو **+30s** للتحكم السريع

### عرض التقدّم

من الناف بار اضغط **📈 تقدمي**، وستجد ٣ تبويبات:

- **🏆 الأرقام القياسية** — كل PRs لكل تمرين (وزن، تكرار، حجم، 1RM)
- **📊 الرسوم البيانية** — تطوّر الوزن، الحجم الأسبوعي، وزن الجسم
- **📜 سجل الجلسات** — كل الجلسات بترتيب زمني مع المدة والحجم

### عرض الإحصائيات السريعة

اضغط الزر العائم 📊 في أسفل يسار الشاشة → **إحصائياتي**

### النسخ الاحتياطي

**مهم جداً:** البيانات محفوظة على متصفح جوالك فقط. لو مسحت بيانات المتصفح أو فقدت الجوال = تخسر التاريخ.

**أوصي بشدّة:**
- اعمل **تصدير** كل أسبوع (الزر العائم → تصدير البيانات)
- احفظ ملف JSON على Google Drive أو iCloud
- لو غيّرت جوال، استخدم **استيراد** لاستعادة البيانات

## 🛠 التطوير المحلي

لاختبار التطبيق قبل النشر:

```bash
# انتقل لمجلد المشروع
cd bulkmode

# شغّل سيرفر محلي (Python موجود في كل أجهزة Mac/Linux)
python3 -m http.server 8000

# أو لو عندك Node.js:
npx serve
```

ثم افتح المتصفح على: `http://localhost:8000`

**ملاحظة:** Service Worker لا يعمل من `file://` — يجب استخدام سيرفر.

## ⚠️ ملاحظات مهمة

- **التخزين محلي فقط:** البيانات على جهازك، ليست على سيرفر. لا حساب، لا تسجيل دخول، لا تتبع.
- **خصوصية كاملة:** لا يوجد analytics، لا cookies، لا اتصال بسيرفرات خارجية (إلا Google Fonts للخطوط — تُحفظ في cache بعد أول تحميل).
- **لا يدعم المزامنة بين أجهزة:** البيانات لا تنتقل تلقائياً بين الجوال واللابتوب. استخدم خاصية تصدير/استيراد JSON يدوياً.

١. عدّل الملف `index.html` على جهازك


## 🧠 البنية التقنية (V5)

- **IndexedDB:** ٧ stores — `workouts`, `sets`, `exercises`, `bodyMetrics`, `prs`, `progressPhotos`, `settings`
- **Migration:** ترحيل تلقائي من LocalStorage عند أول تشغيل
- **Chart.js:** يُحمَّل lazy من CDN واحد (jsdelivr) عند فتح تبويب الرسوم — يُخزَّن في cache بعد أول تحميل
- **Rest Timer:** يعتمد على `Date.now()` بدل `setInterval` للعدّ، يستمر بدقّة حتى مع خمول التاب
- **Notification API + Vibration API:** للتنبيه عند انتهاء الراحة وتحطيم PR
- **Page Visibility API:** للحفاظ على دقّة المؤقت بعد العودة للتاب
- **Web Audio API:** للنقرات الصوتية (تكوين AudioContext عند أول لمسة لتوافق iOS)
- **Vanilla JS:** لا frameworks، لا build، لا modules — كل شيء في `index.html`

راجع `CHANGELOG.md` لتفاصيل التغييرات بين الإصدارات.

## ❓ مشاكل شائعة

**"التطبيق لا يعمل أوفلاين"**  
تأكد أنك زرت الموقع مرة واحدة على الأقل وأنت متصل بالإنترنت — Service Worker يحتاج التشغيل أول مرة ليحفظ كل شيء في cache.

**"خاصية الإضافة للشاشة الرئيسية لا تظهر"**  
- Chrome/Edge على Android: تأكد من إصدار حديث  
- Safari على iOS: استخدم Safari فقط (ليس Chrome على iOS)  
- الموقع يجب أن يكون على `https://` (GitHub Pages يوفر هذا تلقائياً)

**"البيانات اختفت فجأة"**  
على الأغلب مسحت بيانات المتصفح. للأسف لا يمكن استرجاعها بدون نسخة احتياطية JSON. **اعمل تصدير دوري دائماً.**
